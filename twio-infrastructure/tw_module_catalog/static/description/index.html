TW Module Catalog is a powerful tool designed to centralize and manage custom Odoo modules developed within your organization. It automatically syncs with GitHub to provide a comprehensive, searchable catalog of all your internal tools.

What the Module Does

The TW Module Catalog acts as an internal "App Store" for your company's Odoo modules. It periodically scans your GitHub repositories for Odoo modules, parses their metadata (manifests, readmes, index files), and creates a rich dashboard where developers and consultants can discover existing solutions, track versions, and avoid duplicating work.

Why the Module is Useful

This module solves several pain points in multi-repository Odoo development:

  • Discoverability: Stop asking "have we built this before?". Search through all internal modules in one place.
  • Redundancy Detection: Identify similar modules across different repositories using advanced "Cluster ID" hashing.
  • Knowledge Sharing: Access documentation (Readmes and Index files) directly within Odoo without leaving the ERP.
  • Usage Tracking: See which modules are popular and which users have experience with specific tools.
  • Automation: Keeps your catalog up-to-date automatically via scheduled GitHub synchronization.

How Users Can Get Started

Initial Configuration
  1. Ensure the tw_infrastructure module is installed.
  2. Go to Settings > Technical > System Parameters.
  3. Create or update the parameter twio.github_token with a valid GitHub Personal Access Token.
Setup Master Data

If there are repositories that should not be scanned (e.g., standard Odoo, themes, or archived projects):

  1. Navigate to Infrastructure > Configuration > Repository Blacklist.
  2. Add the names of the repositories you want to exclude.
Key Workflows

Browsing Modules:

  • Go to Infrastructure > All Modules.
  • Use the search bar to find modules by name, technical name, or functionality.
  • Group by Siblings to see variations of the same module across different repositories.

Marking Usage:

  • Open a module record.
  • Click the "I have used this module" button to let others know you have experience with it.

Technical Details

List of New Fields and Models

Custom Modules (tw.module.catalog): Infrastructure > All Modules

  • Technical Name: The Odoo technical folder name.
  • Name: The descriptive name from the manifest.
  • Summary / Description: Functional and technical overviews.
  • Version: Current version in the matched repository.
  • Repository Link: Direct link to the code on GitHub.
  • Cluster Group: Identifies identical or highly similar modules.
  • Usage Count: Number of users tracking their experience with this module.

Repository Blacklist (tw.github.repo.blacklist): Infrastructure > Configuration > Repository Blacklist

  • Name: Repository name to exclude from synchronization.
Added Actions and Buttons
  • Sync GitHub Catalog: Orchestrates the synchronization with GitHub (executed via Cron or manual developer action).
  • I have used this module: (Custom Module Form) Mark yourself as a user/owner of this module integration.
  • Remove Blacklisted Repo: (Blacklist Form) Deletes all catalog entries associated with a blacklisted repository.
Module Dependencies
