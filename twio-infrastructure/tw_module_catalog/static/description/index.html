TW Module Catalog is a powerful tool designed to centralize and manage custom Odoo modules developed within your organization. It automatically syncs with GitHub to provide a comprehensive, searchable catalog of all your internal tools.

What the Module Does

The TW Module Catalog acts as an internal "App Store" for your company's Odoo modules. It periodically scans your GitHub repositories for Odoo modules, parses their metadata (manifests, readmes, index files), and creates a rich dashboard where developers and consultants can discover existing solutions, track versions, and avoid duplicating work.

Why the Module is Useful

This module solves several pain points in multi-repository Odoo development:

  • Discoverability: Stop asking "have we built this before?". Search through all internal modules in one place.
  • Redundancy Detection: Identify similar modules across different repositories using advanced "Cluster ID" hashing.
  • Knowledge Sharing: Access documentation (Readmes and Index files) directly within Odoo without leaving the ERP.
  • Usage Tracking: See which modules are popular and which users have experience with specific tools.
  • Automation: Keeps your catalog up-to-date automatically via two specialized background processes.

How Users Can Get Started

Initial Configuration
  1. Ensure the tw_infrastructure module is installed.
  2. Go to Settings > General Settings > Infrastructure.
  3. Configure your GitHub Token. This token is used to authenticate with the GitHub API (requires repo access).
  4. Configure your GitHub Organization. Add the organization name you want to scan for modules.
Setup Master Data

If there are repositories that should not be scanned (e.g., standard Odoo, themes, or archived projects):

  1. Navigate to Infrastructure > Repo Blacklist.
  2. Add the names of the repositories you want to exclude.
  3. Use the "Remove Blacklisted Repo" button to clean up existing entries.
Key Workflows

Browsing Modules:

  • Go to Infrastructure > Catalog > Modules.
  • Use the search bar for global search (Name, Tech Name, Summary, etc.).
  • Group by Siblings to see variations of the same module across different repositories.

Checking Repositories & Archives:

  • Go to Infrastructure > Catalog > Repositories.
  • Open a repository to see its current modules.
  • Use the "Archived Modules" tab to see modules that were previously detected but have since been removed from the repository.

Marking Usage:

  • Open a module record.
  • Click "I have used this module" to track your experience and help others find experts.

Automated Sync & Archival

The module maintains an up-to-date catalog using two essential background Cron jobs and a robust archival system:

  • Discovery (Queue Filler):
    Scans the GitHub organization daily to identify new repositories or changes. It populates a synchronization queue with pending updates.
  • Smart Archiving:
    If a module is removed from a GitHub repository, the system automatically archives it instead of deleting it. This preserves historical relations and usage data.
  • Queue Processor (Worker):
    Processes pending updates in efficient bursts, fetching manifests and documentation without overloading your server.

Note: Synchronization typically respects a 12-hour threshold to avoid redundant API calls.

Technical Details

List of New Fields and Models

Custom Modules (tw.module.catalog):

  • Technical Name: Odoo technical folder name.
  • Name: Descriptive name from manifest.
  • Cluster Group: Groups identical/similar modules using 5-pillar hashing.
  • Usage Count: Number of users who have marked this module as used.
  • Repository Link: Direct link to the source code.
  • Readme / Description View: Rich text/HTML previews of module documentation.
Added Actions and Buttons
  • action_toggle_usage: ("I have used this module") Toggles the current user's experience tracking for a module.
  • action_discovery_cron: Manual trigger for the GitHub Discovery process.
  • action_process_queue_cron: Manual trigger for processing the sync queue.
Module Dependencies
  • tw_infrastructure
  • mail (Chatter and Activities enabled)
