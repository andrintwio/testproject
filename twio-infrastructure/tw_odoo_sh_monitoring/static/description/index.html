Odoo.sh Monitoring is a comprehensive module that monitors and manages Odoo.sh repositories, branches, and user access. It automatically synchronizes data from Odoo.sh, tracks build statuses, manages repository users, and provides notifications for branch changes.

What the Module Does

This module provides comprehensive monitoring and management of Odoo.sh infrastructure:

  • Repository Monitoring: Tracks all Odoo.sh repositories, their branches, and build statuses
  • Automatic Synchronization: Hourly cron job automatically syncs repository and branch data from Odoo.sh
  • Branch Management: Monitors branch build statuses with visual indicators (🟢 success, 🟡 warning, 🔴 failed, ⚪ no build)
  • User Access Management: Tracks and manages repository users with their permissions (Admin, Developer, Tester)
  • Branch Blocking: Allows blocking branches for specific users with automatic notifications when changes occur
  • Integration: Links Odoo users and employees with GitHub users and repository access

Why the Module is Useful

This module provides significant value for managing Odoo.sh infrastructure:

  • Centralized Monitoring: View all repositories and branches in one place without switching to Odoo.sh interface
  • Real-time Status: Get immediate visibility into build statuses with color-coded indicators
  • Automated Sync: No manual data entry - repositories and branches are automatically synchronized hourly
  • User Management: Easily manage repository access and permissions directly from Odoo
  • Change Notifications: Receive automatic email notifications when blocked branches are updated
  • Employee Integration: Link repository access to employees for better visibility of who has access to what
  • Historical Tracking: Track build history, commit messages, and pusher information for each branch

How Users Can Get Started

Initial Configuration

Before using the module, you need to configure Odoo.sh connection settings:

  1. Go to Settings > Infrastructure > Odoo.sh Monitoring
  2. Enter your Odoo.sh Session ID:
    • Log in to Odoo.sh (https://www.odoo.sh)
    • Open your browser's developer tools (F12)
    • Go to Application/Storage > Cookies
    • Copy the value of the session_id cookie
    • Paste it into the Session ID field
  3. Enter your Default Project ID (default: 359937)
  4. Click Save

Note: The Session ID may expire periodically. If synchronization fails, update the Session ID.

Setup Master Data

1. Synchronize Repositories:

  1. Go to Infrastructure > Odoo.sh Repositories
  2. Click the "Synchronize Odoo.sh Repositories" button
  3. Wait for the synchronization to complete (this may take a few minutes)
  4. All repositories, branches, and users will be automatically imported

2. Link Users to GitHub:

  1. Go to Settings > Users & Companies > Users
  2. Open a user's form
  3. Enter the user's GitHub User name in the GitHub field
  4. Save the user

3. Assign Security Groups:

  1. Go to Settings > Users & Companies > Groups
  2. Assign appropriate groups to users:
    • Infrastructure Admin: Full access (can delete repositories, manage all settings)
    • Infrastructure Developer: Can view and edit branches, manage users
    • Infrastructure Consultant: Read-only access
Key Workflows

1. Viewing Repository Status:

  1. Go to Infrastructure > Odoo.sh Repositories
  2. View the status column (🟢/🟡/🔴/⚪) to see overall repository health
  3. Click on a repository to see details, branches, and users

2. Monitoring Branches:

  1. Go to Infrastructure > Odoo.sh Branches
  2. View all branches grouped by repository
  3. Check the status column for build status
  4. Use filters to find branches by status (Success, Failed)
  5. Click on a branch to see build details, commit messages, and history

3. Blocking a Branch:

  1. Go to Infrastructure > Odoo.sh Branches
  2. Open the branch form
  3. Select a user in the "Blocked By" field
  4. Optionally set a "Blocked Until" date
  5. Save - the user will receive email notifications if the branch is updated

4. Managing Repository Users:

  1. Go to Infrastructure > Odoo.sh Repositories
  2. Open a repository
  3. Go to the "Users" tab
  4. Add new users by clicking "Add a line" and selecting a user
  5. Set permissions (Admin, Developer, Tester)
  6. Changes are automatically synced to Odoo.sh

5. Viewing Employee Repository Access:

  1. Go to Employees app
  2. Open an employee's form
  3. Click the "Repository Access" button (if the employee has a linked user)
  4. View all repositories the employee has access to

Technical Details

List of New Fields and Models

New Models:

  • Odoo.sh Repository (Infrastructure > Odoo.sh Repositories)
    • Repository Name - Name of the repository
    • Odoo.sh Repository ID - Internal ID from Odoo.sh
    • Owner - Repository owner
    • Odoo Branch - Odoo version branch
    • Project Name - Project name in Odoo.sh
    • Branch Count - Number of branches (computed)
    • Status - Visual build status indicator (🟢/🟡/🔴/⚪/❓)
    • Last Sync - Last synchronization timestamp
    • Sync Status - Status message from last sync
  • Odoo.sh Branch (Infrastructure > Odoo.sh Branches)
    • Branch Name - Name of the branch
    • Repository - Link to parent repository
    • Stage - Branch stage (Production, Staging, Dev)
    • Status - Visual build status indicator (🟢/🟡/🔴/⚪)
    • Last Build Status - Status of last build
    • Last Build Result - Result of last build
    • Last Build Commit Message - Commit message from last build
    • Last Build Commit Author - Author of last commit
    • Last Build Date - Date of last build
    • Last Pusher Name - Person who triggered the build
    • Tracking Count - Number of build trackings
    • Blocked By - User to notify on branch changes
    • Block Date - Date when branch was blocked
    • Blocked Until - Date until which branch is blocked
    • Last Sync - Last synchronization timestamp
  • Repository User (Accessible via Repository form > Users tab)
    • Repository - Link to repository
    • User - Link to Odoo user
    • User Name - Name of the user (computed)
    • GitHub User - GitHub username
    • Permission - Access level (Admin, Developer, Tester)

Extended Models:

  • Users (Settings > Users & Companies > Users)
    • GitHub User - GitHub username field (visible in user form)
  • Employees (Employees app)
    • Repository Access Count - Number of repositories the employee has access to (computed, visible as button)
  • Settings (Settings > Infrastructure > Odoo.sh Monitoring)
    • Odoo.sh Session ID - Session ID for API authentication
    • Default Project ID - Default project ID for monitoring
Added Actions and Buttons

Server Actions:

  • Synchronize all Odoo.sh Repositories (Repository list view header)
    • Model: Odoo.sh Repository
    • Location: Infrastructure > Odoo.sh Repositories > Header button
    • Functionality: Fetches all repositories, branches, and users from Odoo.sh and synchronizes the data. Removes branches and users that no longer exist in Odoo.sh. Shows a notification with sync results.

Object Actions:

  • View Repository Users (Employee form)
    • Model: Employee
    • Location: Employees app > Employee form > "Repository Access" button
    • Functionality: Opens a filtered view of Repository Users for the employee's linked user account.

Automatic Actions (Cron Jobs):

  • TW: Sync Odoo.sh Repositories
    • Runs: Every hour
    • Functionality: Automatically synchronizes all repositories, branches, and users from Odoo.sh
  • TW: Check Blocked Until Reminders
    • Runs: Daily
    • Functionality: Checks branches with expired "Blocked Until" dates and sends reminder emails to the blocked user

Automatic Notifications:

  • Branch Change Notification
    • Trigger: When a blocked branch receives a new build/push
    • Recipient: User assigned in "Blocked By" field
    • Content: Details about the change (pusher, commit message, date)
  • Block Reminder Notification
    • Trigger: When "Blocked Until" date is reached (checked daily)
    • Recipient: User assigned in "Blocked By" field
    • Content: Reminder to review and update the block status
